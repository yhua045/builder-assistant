## Summary

This PR completes the invoicing module refinements to improve consistency and simplify the user experience.

## Changes

### Invoice Entity & Schema Updates
- Updated Invoice entity and schema to include issuer_id field for persistent vendor tracking
- Added issuer_id: text('issuer_id') column to the invoices table via migration 0017_invoices_issuer_id
- This enables immutable invoice auditing and eliminates the need for Contact lookups on every invoice read

### Vendor Tracking Improvements
- Replaced manual unmanaged vendor tracking with ContractorLookupField across both InvoiceForm and ReceiptForm
- The reusable ContractorLookupField component provides autocomplete and contact creation affordance
- Vendors are now persisted as issuer_id on the invoice entity

### Migration Fixes
- Fixed duplicate Drizzle migration: removed duplicate ALTER TABLE invoices ADD COLUMN pending_payments
- Verified migration system applies migrations only once per app boot
- Added migration 0017_invoices_issuer_id to persist the new vendor ID field

### UI Flow Simplification
- Removed arbitrary hidden interactions from InvoiceScreen (Enter Details vs Pick PDF routing)
- InvoiceForm now explicitly exposes tabbed interface for PDF upload and manual entry
- Users can directly choose their invoice entry method without hidden conditional routing

### Code Quality
- Fixed linting errors in InvoiceForm.tsx: prefixed unused setters with underscore
- All tests passing: 757 tests, 7 skipped, 0 failures
- TypeScript compilation clean
