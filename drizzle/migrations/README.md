# Drizzle Migrations

This directory contains all database migrations generated by Drizzle Kit.

## Migration workflow

### 1. Generate a migration
After making changes to the schema in `src/infrastructure/database/schema.ts`, run:
```bash
npx drizzle-kit generate
```

This will create a new SQL migration file in this directory.

### 2. Apply migrations
Migrations are automatically applied when the app starts via `initDatabase()` in `connection.ts`.

### 3. View current schema
To see the current database schema:
```bash
npx drizzle-kit introspect
```

### 4. Push changes directly (development only)
To push schema changes directly without generating migrations:
```bash
npx drizzle-kit push
```

⚠️ **Warning**: This should only be used in development. Use migrations for production.

## Migration files

Migration files are named with a sequential number and a random identifier:
- `0000_initial_schema.sql` - Initial database schema
- `0001_add_user_preferences.sql` - Example of adding a new table
- `0002_alter_projects_table.sql` - Example of modifying existing table

Each migration is tracked in the `__drizzle_migrations` table to ensure it only runs once.
